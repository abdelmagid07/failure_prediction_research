# Failure Prediction Research

Research workspace for **latent failure prediction in long-horizon language
agents**: does a frozen "value axis" in Qwen3-8B's residual stream transfer from
single-turn data to multi-step SWE-bench agent trajectories and separate eventual
success from failure? Target: a NeurIPS mechanistic-interpretability workshop.

## The working codebase

Active experiments and paper work live at the repo root:

| Path | Description |
|------|-------------|
| `stage1/` | ICRL → activation extract → value axis + AUROC gate (**done**, 0.87 AUROC) |
| `stage2/` | Trajectory generate → ingest → projection → readability analyses (**pipeline built**) |
| `docs/` | Setup, method, onboarding, analyses, walkthrough |
| `tests/integration/` | Offline wiring tests (no GPU) |
| `reference/latent_failure_prediction/` | **Frozen, read-only** snapshot of the original pipeline. Baseline for comparison — do not edit. |

Trajectories are generated by **mini-swe-agent** on real SWE-bench (primary) or
the local **devbugs** harness (no-Docker dev smoke tests). See
[HANDOFF.md §4](HANDOFF.md) for the recent SWE-agent → mini-swe-agent migration.
