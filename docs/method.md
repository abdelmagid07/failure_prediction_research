# Method

## Research question

Does the value axis from Jiang et al. (2026) persist and track eventual success or failure when projected onto **long-horizon, on-policy agent trajectories**?

This is a transfer test: the axis is constructed on single-turn ICRL data, frozen, then read off multi-step SWE-bench (or mini-agent) trajectories generated by the same model (Qwen3-8B).

**Thinking-mode note (decision 2026-07-17).** Stage 2 now generates and projects trajectories with Qwen3 **thinking ON**. The frozen axis, however, was built in Stage 1 with thinking OFF (the ICRL construction uses literal `<thinking>...</thinking>` text tags with the chat template's native thinking mode disabled). This makes the Stage 2 result an explicit **cross-mode transfer** claim — a thinking-off axis read off thinking-on trajectories — which the paper must state plainly. The axis is not rebuilt.

## Anchor result

- Paper: Jiang, Kauvar, Lindsey (2026), arXiv [2606.17056](https://arxiv.org/abs/2606.17056)
- Model: **Qwen3-8B**
- Construction: difference-in-means of residual-stream activations before vs. after criterion discovery
- Reported held-out AUROC > 0.9; strongest layers ~21–22

## Pipeline stages

1. **Stage 1 — Reconstruct and verify.** Build the value axis from ICRL conversations; gate on held-out AUROC (default threshold 0.93 at L21/L22). Freeze `value_axis.npy`.
2. **Stage 2 — Trajectory projection.** On-policy agent trajectories → per-step projections with `reasoning` vs `tool_output` tags → readability analyses.
3. **Stage 3 — Trajectory evolution** (planned). Projection vs. relative position; elicited confidence baseline.
4. **Stage 4 — Localization** (planned). Layer-wise transfer analysis.

## Locked decisions

| Decision | Value |
|----------|-------|
| Model | `Qwen/Qwen3-8B` |
| Thinking mode | Stage 2: `enable_thinking=True` (generation + projection). Stage 1 axis was built `enable_thinking=False` → cross-mode transfer (see note above) |
| Projection layer | 21 (verify from Stage 1 manifest) |
| Decoder layers | 36 (indices 0–35) |
| Trajectories | On-policy only (same model that generated them) |
| Read position | Last token of assistant output at each step |
| Axis | Frozen after Stage 1; no refit on agent data |
| Benchmark | SWE-bench **Verified** (`test` split) everywhere (decision 2026-07-18); mini-agent env for development |
| Sampling | Qwen3 thinking recipe: `temperature=0.6`, `top_p=0.95`, `top_k=20` (decision 2026-07-18) |
| Baseline | Majority-class, not 0.5 chance |

## Null results

Two distinct outcomes must not be conflated:

1. **Genuine no-signal:** measurement was sensitive enough; the axis does not separate outcomes in the agentic regime.
2. **Noise-dominated:** within-class variance is too high to interpret a null; this is a measurement limitation, not evidence of absence.

Report separation-to-noise alongside any null. A publishable negative result requires a demonstrated noise floor.

## Development preset

`--preset dev` uses Qwen-local ICRL, a 0.75 gate threshold, and separate artifact files (`value_axis_proxy.npy`, etc.). Use for pipeline development; the default preset is the faithful reproduction path.
