# Method

Source of truth for the research design: [METHOD.tex](../METHOD.tex). This page
is the short operational summary used by the codebase.

## Research question

Does the value axis from Jiang et al. (2026) persist and track eventual success
or failure when projected onto **long-horizon, on-policy agent trajectories**?

This is a transfer test: the axis is constructed on single-turn ICRL data,
frozen, then read off multi-step SWE-bench Verified trajectories generated by
the same model (Qwen3-8B).

**Thinking-mode note (decision 2026-07-17).** Stage 2 generates and projects
with Qwen3 **thinking ON**. The Stage 1 axis was built thinking OFF → explicit
**cross-mode transfer**. The axis is not rebuilt.

**Paper vs codebase stage numbering.** METHOD.tex includes a single-turn coding
control (paper Stage 2) that is **deferred**. Codebase stages therefore map as:

| METHOD.tex | Codebase |
|------------|----------|
| Stage 1 — axis reconstruction | `stage1/` |
| Stage 2 — single-turn coding control | deferred |
| Stage 3 — transfer to agentic trajectories | `stage2/` (this package) |
| Stage 4 — verbalized confidence | `stage2/elicit/` + `analyze/internal_vs_elicited` |
| Stage 5 — fitted-probe reference | `stage2/probes/` |

## Anchor result

- Paper: Jiang, Kauvar, Lindsey (2026), arXiv [2606.17056](https://arxiv.org/abs/2606.17056)
- Model: **Qwen3-8B**
- Construction: difference-in-means of residual-stream activations
- Reported held-out AUROC > 0.9; strongest layers ~21–22

## Pipeline stages (codebase)

1. **Stage 1 — Reconstruct and verify.** Build the value axis from ICRL;
   gate on held-out AUROC within **0.03** of the published ~0.90 (floor 0.87)
   at L21/L22. Freeze `value_axis.npy`. Primary layer = highest held-out AUROC
   in *our* reconstruction, fixed before agentic data.
2. **Stage 2 — Agentic transfer.** 60 Verified tasks × 5 seed-only rollouts
   (selection criteria TBD). Per-step readout is **mean cosine over
   agent-generated tokens** `G_t` (METHOD.tex Eq. 1); final-token cosine is a
   robustness column. Final-step AUROC + position-bin AUROC with task-level BCa
   CIs and block permutation.
3. **Stage 3 — Verbalized confidence.** Post-hoc fork/replay elicitation of
   P(success)∈[0,1] per prefix; compare to internal `proj_mean` by position.
4. **Stage 4 — Fitted probes.** Logistic probes on mean-pooled activations,
   task-level CV, full layer × position grid (never just the max).

## Null results

Two distinct outcomes must not be conflated:

1. **Genuine no-signal:** measurement was sensitive enough; the axis does not
   separate outcomes in the agentic regime (final-step CI includes 0.5 *and*
   the positional sweep agrees).
2. **Noise-dominated / infra-contaminated:** too few resolved labels, or
   unresolved dominated by format/infra failures rather than capability.

Report majority baseline, CI, permutation p, and exit_status taxonomy alongside
any null.

## Development preset

`--preset dev` uses Qwen-local ICRL, a 0.75 gate threshold, and separate
artifact files (`value_axis_proxy.npy`, etc.). Use for pipeline development; the
default preset is the faithful reproduction path.
